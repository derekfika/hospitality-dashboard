# CPU Production Dashboard

A chef-first Google Apps Script dashboard for Fika’s central production unit.
It scans hospitality and delivery events from configured Google Calendars,
reads attached Google Docs and Microsoft Office quotes, caches the resulting
production data in Google Sheets, and presents a multi-site operational view.

Current build: `2026.06.20.14`

## Core capabilities

- Monday-to-Friday calendar with site, delivery time, pax and service
- Multi-site, service, day, category, updated-only and completion filters
- Categorised production board with totals, site splits and booking drill-down
- Run sheet, Dietary, Deliveries and full-screen Kitchen views
- Prep progress tracking with persistent Prepped/Done flags
- Quick-prep controls on calendar and production records
- Changed-booking alerts showing pax, time, location, product and dietary changes
- Needs Attention panel for missing or unreadable production information
- Dedicated scheduled-delivery records and calendar day badges
- Next-week bookings and pax forecast
- Incremental scanning that skips unchanged attachments
- Persistent user filters and view preferences
- Keyboard navigation, loading skeletons and urgent-order highlighting
- Print styling retained in code for possible future chef requirements

## Calendar and attachment workflow

The scanner supports multiple calendars through `CALENDARS_JSON`. Events with
hospitality attachments or recognised hospitality titles are treated as
bookings. Events containing “Delivery Scheduled”, “Delivery Reminder” or
collection wording are stored separately in the `CPU Deliveries` sheet.

For bookings, the scanner:

1. Resolves the site from the event creator/organiser email.
2. Finds quote and booking-form attachments or Drive links in the description.
3. Reads Google Docs, DOCX, Sheets/XLSX and Slides/PPTX content.
4. Extracts dates, times, pax, products, notes and dietary requirements.
5. Stores the normalised booking in `CPU Orders`.
6. Skips attachment parsing on later scans when the event has not changed.

The quote parser contains regression coverage for the different legacy quote
layouts currently used by Fika sites. The long-term integration should replace
this with structured booking JSON generated by each site dashboard.

## Sheets

`setupCpuDashboard()` creates:

- `CPU Settings`
- `CPU Orders`
- `CPU Deliveries`
- `CPU Scan Log`

The settings sheet includes:

- `CALENDARS_JSON`
- `SITES_JSON`
- `PRODUCT_CATEGORIES_JSON`
- Scan lookback/lookahead settings

Product categories are editable without changing code. Run
`installCpuProductCategories()` to restore the built-in category schema.

## Setup and deployment

1. Create a Google Apps Script project attached to the CPU data spreadsheet.
2. Copy the files from this folder or deploy with the included `.clasp.json`.
3. Enable the advanced Calendar API and Drive API.
4. Run `setupCpuDashboard()` and approve permissions.
5. Configure calendar IDs in `CPU Settings > CALENDARS_JSON`.
6. Ensure the deployment account can open all attached quote files.
7. Run the first calendar scan.
8. Optionally run `installCpuRefreshTrigger()` for hourly background scans.
9. Create a new Apps Script deployment version and open its `/exec` URL.

The client and server build numbers must match. A mismatch banner means an older
deployment URL or cached page is being used.

## Site ownership

The built-in site directory maps event-owner email addresses to site names,
codes and colours. This is the authoritative site assignment because the event
owner nearly always represents the delivery destination. `SITES_JSON` can
override or extend this directory.

## File permissions

The Apps Script deployment account must have permission to open attached files.
A link visible on a shared calendar does not automatically grant Drive access.
Sites should share source files with the CPU scanner account or its Google Group.

## Operational notes

- Prepped status is stored separately and survives calendar rescans.
- Manual quote re-reading updates the editor and dashboard immediately.
- Changed-booking alerts remain visible for seven days.
- Kitchen mode refreshes cached sheet data every two minutes.
- The Print button is currently hidden, but `window.print()` and print CSS remain.

## Testing

Run `runCpuPureTests()` in Apps Script. The test suite covers parser edge cases,
site mapping and legacy quote formats.
